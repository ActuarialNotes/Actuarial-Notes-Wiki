An insurer is planning to revise burglar alarm and deductible rating plan factors for its Homeowners program. Given the following generalized linear model output:

**Burglar Alarm:**

| Burglar Alarm | GLM Prediction | −2 Standard Errors | +2 Standard Errors | Policies |
|---|---|---|---|---|
| None | 1.00 | — | — | 320,000 |
| Local Alarm | 0.98 | 0.950 | 1.010 | 27,500 |
| Central Reporting | 0.86 | 0.730 | 0.990 | 2,500 |

**Deductible Amount:**

| Deductible | GLM Prediction | −2 Standard Errors | +2 Standard Errors | Policies |
|---|---|---|---|---|
| $250 | 1.75 | 1.60 | 1.90 | 2,700 |
| $500 | 1.10 | 1.05 | 1.15 | 87,000 |
| $1,000 | 1.00 | — | — | 150,000 |
| $2,500 | 0.95 | 0.90 | 1.00 | 60,000 |
| $5,000 | 0.85 | 0.80 | 0.90 | 50,100 |
| $7,500 | 1.25 | 0.90 | 1.60 | 150 |
| $10,000 | 0.40 | 0.00 | 0.80 | 50 |

Propose revised burglar alarm and deductible rating plan factors. Document the relevant analysis and rationale to support the proposal.

### Answer

**Burglar Alarm — Proposed factors: None = 1.00, Local Alarm = 1.00, Central Reporting = 1.00**

**Deductible — Proposed factors:**

| Deductible | Proposed Factor |
|---|---|
| $250 | 1.50 |
| $500 | 1.10 |
| $1,000 | 1.00 |
| $2,500 | 0.95 |
| $5,000 | 0.85 |
| $7,500 | 0.75 |
| $10,000 | 0.65 |

### Explanation

**Burglar Alarm Analysis**

*Local Alarm (27,500 policies):* The GLM prediction is 0.98, very close to the base (None = 1.00). The 95% confidence interval (±2 standard errors) spans 0.950 to 1.010, which includes 1.00. Because the confidence interval encompasses the base level relativity, the Local Alarm factor is **not statistically significantly different from None**. The volume is also relatively low.

*Central Reporting (2,500 policies):* The GLM prediction is 0.86, which would suggest a discount relative to None. However, there are only 2,500 policies in this category — very sparse data. The confidence interval is extremely wide (0.730 to 0.990), reflecting high uncertainty.

**Recommendation — Burglar Alarm:** Do not implement the burglar alarm variable at this time. Both non-base levels have insufficient data and/or confidence intervals that include 1.00. Select **1.00 for all burglar alarm levels** until more data is available to confirm a statistically significant and practically meaningful difference.

---

**Deductible Analysis**

*$250 (2,700 policies):* GLM prediction = 1.75, confidence interval 1.60–1.90. The data is sparse (only 2,700 policies), introducing more uncertainty. A judgment factor of **1.50** is selected — directionally consistent with the GLM, but pulled toward a more moderate value given the limited data.

*$500 (87,000 policies):* GLM prediction = 1.10, confidence interval 1.05–1.15. Large volume, tight confidence interval. The GLM prediction is reliable. Select **1.10** (use indicated factor).

*$1,000 (150,000 policies):* Base class with largest volume. Select **1.00** (base).

*$2,500 (60,000 policies):* GLM prediction = 0.95, confidence interval 0.90–1.00. Reasonable volume, tight interval, directionally sensible (higher deductible → lower relativity). Select **0.95** (use indicated factor).

*$5,000 (50,100 policies):* GLM prediction = 0.85, confidence interval 0.80–0.90. Good volume, tight interval. Select **0.85** (use indicated factor).

*$7,500 (150 policies):* GLM prediction = 1.25 — a reversal. This is higher than the $5,000 factor (0.85), which is counterintuitive: a higher deductible should produce a lower (or at most equal) factor. Only 150 policies — very sparse data with a very wide confidence interval (0.90–1.60). The reversal and wide error band are almost certainly the result of sparse data. Select the midpoint between $5,000 and $10,000 judgmentally: **(0.85 + 0.65) / 2 = 0.75**.

*$10,000 (50 policies):* GLM prediction = 0.40, confidence interval 0.00–0.80. Only 50 policies — extremely sparse. The confidence interval touching 0.00 indicates virtually no information. The indicated factor (0.40) is implausibly low given the trend of other deductibles. Select **0.65** judgmentally, continuing the downward monotone progression from $5,000 at a reasonable pace.

**Deductible summary:** Factors for $500 through $5,000 use GLM indicated factors (good data, tight confidence intervals, directionally sensible). The $250 level is adjusted downward from the GLM due to sparse data. The $7,500 and $10,000 levels are set judgmentally because the GLM output is unreliable due to extremely sparse data and/or counterintuitive results; the selected factors maintain a monotone decreasing pattern consistent with higher deductibles implying lower coverage cost.

### Examiner Report

In general, the response to this question was poor. Many candidates recognized the small data volume but incorrectly went about combining alarm types or deductibles into one category. This was often accompanied by a calculation of a proposed factor by weighting the GLM output. Time was unnecessarily lost by this calculation. Another common error was that candidates recognized unintuitive output that seemed to be the result of sparse data but yet still proposed to select the predicted factor.

The key skills tested were:

1. **Evaluating statistical significance** using confidence intervals — specifically, recognizing when the interval includes the base (1.00) and when it is so wide as to render the GLM output unreliable.
2. **Recognizing reversals** that violate expected monotone relationships (e.g., $7,500 deductible factor higher than $5,000) and attributing them to sparse data rather than real phenomena.
3. **Applying judgment** to set factors that are directionally sensible, consistent with the data where credible, and blend smoothly where data is sparse or counterintuitive.
